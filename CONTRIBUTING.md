# Contributing

Keep the repository useful as source, tests, configuration, and concise current
documentation. Git history is the archive for task evidence and old reports.

## Commit To Git

- Application source code.
- Tests and smoke scripts.
- Meaningful configuration templates.
- Architecture changes.
- Setup or user-facing documentation changes.
- Major milestone state.
- Important architecture decisions that affect future implementation.

## Keep Local Or Ignored

- Runtime logs.
- Install logs.
- Package snapshots.
- Conda or pip snapshots.
- Generated reports.
- Daily AI task notes.
- Command transcripts.
- Smoke output.
- Temporary WAV files.
- Generated audio.
- Model weights.
- Datasets.
- Pytest temporary folders.
- Local environment evidence.
- External voice-lab reports.
- Secrets and `.env` files.

## Documentation Update Rules

- Update `README.md` only for project introduction, setup, navigation, or major
  user-facing changes.
- Update `docs/ARCHITECTURE.md` only when architecture, data flow, ownership,
  or module boundaries change.
- Update `docs/CURRENT_STATE.md` for milestone state, active blockers, and next
  actions.
- Update `docs/ROADMAP.md` for strategic direction, not task-by-task history.
- Update `docs/SECURITY.md` for stable boundaries and periodic security changes,
  not every task audit.
- Keep task-level evidence in assistant final reports or local ignored notes.
- Do not commit external lab manifests or generated probe reports.
