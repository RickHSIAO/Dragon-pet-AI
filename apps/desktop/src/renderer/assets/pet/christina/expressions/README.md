## Generation Script

`christina_neutral.png` was generated by:

```powershell
python apps/desktop/scripts/create-christina-neutral-asset.py
```

Crop: cols 59-982, rows 0-843 (top 55% of 1024x1536 reference), padded to
924x924, scaled to 512x512 with Lanczos. Reproduces exactly from the reference
PNG.

The user-provided `focused`, `happy`, `proud`, and `annoyed` files are larger
than the ideal 512x512 face/bust spec. They are valid RGBA PNG assets and load
through the renderer PNG path, but future visual QA may crop/normalize them to
the neutral 512x512 framing if the 80x80 display reads too small.

TASK-092 80x80 visual QA:

Preview file:
`../expression-80x80-preview.html`

| Mood | Alpha bbox | Rendered image in 80x80 | Face/expression readability | QA verdict |
|---|---|---:|---|---|
| `neutral` | x18-y29 to x511-y491 | 80.0x80.0 | clear face, horns, and expression | acceptable baseline |
| `focused` | x65-y17 to x988-y1521 | 53.3x80.0 | face and expression too small | normalize recommended |
| `happy` | x0-y0 to x1029-y1526 | 54.0x80.0 | face and smile too small; possible edge fringe | normalize recommended |
| `proud` | x71-y8 to x982-y1532 | 53.3x80.0 | face and smug expression too small | normalize recommended |
| `annoyed` | x0-y0 to x1028-y1527 | 53.9x80.0 | face and annoyed expression too small; possible edge fringe | normalize recommended |

Conclusion: the issue is image framing, not renderer loading. Keeping the
current assets is functional, but the full-body moods do not match the neutral
face/bust crop and read too small in the 80x80 pet display. Preferred next step:
create normalized 512x512 face/bust derivatives from the existing user-provided
source PNGs, without overwriting the originals until visual QA approves them.
Changing the pet display container to 96x96 or 128x128 would make the full-body
assets somewhat more legible, but it would also change layout and still would
not match neutral framing; normalization is the smaller, cleaner fix.
