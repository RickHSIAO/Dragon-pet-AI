# Ollama Runtime Smoke Checklist

## Purpose

This checklist is for TASK-075 - Ollama Runtime Smoke Check.

TASK-074 does not run live Ollama smoke. TASK-074 only strengthens mocked provider contract tests and prepares this checklist.

## Preconditions

- Ollama installed
- `ollama list` shows `qwen3:8b`
- Ollama server running at `http://localhost:11434`
- No Anthropic/OpenAI API key required
- No external provider call expected
- Backend branch clean / committed

## Recommended Env Vars

PowerShell examples:

```powershell
$env:LLM_PROVIDER_ENABLED="true"
$env:LLM_CHAT_ENABLED="true"
$env:LLM_PROVIDER_NAME="ollama"
$env:LLM_MODEL="qwen3:8b"
$env:OLLAMA_BASE_URL="http://localhost:11434"
$env:OLLAMA_KEEP_ALIVE="10m"
$env:OLLAMA_TIMEOUT_SECONDS="30"
```

## Start Commands

```powershell
ollama list
```

Optional warm-up:

```powershell
ollama run qwen3:8b
```

Backend start:

```powershell
cd F:\RickHSIAO\Python\dragon-pet-ai\backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --port 8000
```

`/chat` smoke request:

```powershell
$body = @{
  message = "Say one short sentence as Christina."
  use_memory = $false
} | ConvertTo-Json -Compress

Invoke-RestMethod `
  -Method POST `
  -Uri http://localhost:8000/chat `
  -ContentType "application/json; charset=utf-8" `
  -Body $body
```

## Expected /chat Response

- HTTP 200
- Schema remains `reply / mood / source`
- `source` expected `llm_local` or current implemented source value
- Reply is generated by local `qwen3:8b`
- No API key involved

## Safety Checks

- No request to Anthropic / OpenAI
- No API key in env needed
- No API key in logs
- No raw prompt logging
- No memory context unless memory gates are explicitly enabled
- No external network request except localhost
- If Ollama is not running, safe error category expected

## Failure Cases

- Ollama not running
- Model not pulled
- Timeout
- Invalid model name
- UTF-8 encoding issue
- VRAM/RAM slow first load
